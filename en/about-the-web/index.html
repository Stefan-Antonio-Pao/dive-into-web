Describe what happens when you visit bilibili.com?

When you enter "bilibili.com", your browser performs a DNS lookup to find the IP address of Bilibili's server. Once the address is found, the browser sends an HTTP or HTTPS request to the server, and the server returns the webpage, which is then rendered by the browser.

What are the components of a web page document?

A web page document consists of a DOCTYPE declaration, HTML elements (including the head and body), scripts, and stylesheets.

What is the World Wide Web?

The World Wide Web is an information system where documents and resources are identified by URLs and can be accessed by users through browsers.

Analyze d2w.wkusci.com, what is the TLD (Top-Level Domain) and SLD (Second-Level Domain)?

The TLD is .com, and the SLD is wkusci.

How does DNS work?

DNS is like the phone book of the internet, translating domain names into IP addresses, allowing browsers to access servers.

Analyze https://d2w.wkusci.com:80/blogs/have-a-nice-day/?invitation=4590098&recommend=232423#have-fun

https is the protocol or schema for transmitting data.

d2w.wkusci.com:80 is the authority, where d2w.wkusci.com is the domain and 80 is the port. Port 80 is the default for HTTP, and 443 for HTTPS. If your backend server provides services on port 6666, https://d2w.wkusci.com:6666/user/?id=12321 would be a complete URL where you access services provided on port 6666 of d2w.wkusci.com with parameters id=12321.

The scheme and authority are separated by ://. The colon separates the scheme from the next part of the URL, and // indicates the next part is the authority. One example of a URL without authority is a mail client mailto:foobar, which includes a scheme but no authority.

After the domain name is the path /blogs/have-a-nice-day/, ending with a slash (required when the URL represents a directory).

Following the path, the URL includes ?invitation=4590098&recommend=232423, which provides query parameters. The #have-fun at the end is a fragment identifier for navigating within the page.

What is a static server and what is a dynamic server?

A static server refers to a server that only provides static resources such as HTML, CSS, JavaScript, images, videos, etc. A dynamic server, on the other hand, can return different content based on the request. We call it "dynamic" because the application server updates the hosted files, such as html, before sending them to your browser via the HTTP server.
