描述你的浏览器访问bilibili.com发生了什么？

当你输入 "bilibili.com" 时，浏览器会进行 DNS 查询，找到 Bilibili 服务器的 IP 地址。找到后，浏览器向服务器发送 HTTP 或 HTTPS 请求，服务器返回网页，浏览器渲染显示。

一个网页文档由哪些部分组成？

网页文档由 DOCTYPE 声明、HTML 元素（包括头部和主体内容）、脚本和样式表组成。

什么是万维网？

万维网是一个信息系统，通过 URL 标识文档和资源，用户可以通过浏览器访问这些资源。

DNS 如何工作？

DNS 就像是互联网的电话簿，它将域名翻译成 IP 地址，使得浏览器能够访问服务器。

分析 https://d2w.wkusci.com:80/blogs/have-a-nice-day/?invitation=4590098&recommand=232423#have-fun

https 是传输数据的协议或方案 schema。

d2w.wkusci.com:80 是 权威，authority。 d2w.wkusci.com 是域名，80是端口。80为 http 默认端口，443为 https 默认端口，如果你的后端服务器通过6666端口提供服务 那么 https://d2w.wkusci.com:6666/user/?id=12321 就是一个完整的 URL,你访问的是 d2w.wkusci.com 的 6666 端口提供的服务，参数为 id=12321

方案和权威之间的分隔符是 ://。冒号将方案与 URL 的下一部分分隔开，而 // 表示 URL 的下一部分是权威。 不使用权威的 URL 示例之一是邮件客户端mailto:foobar。它包含方案，而不使用权威的部分。因此，冒号后没有跟着两个斜杠，并且只充当方案和邮件地址之间的分隔符。

/blogs/have-a-nice-day 是我们访问的资源路径，由服务器的 API 端口负责接收和处理请求。

? 后面是提供给 Web 服务器的额外参数。这些参数是用 & 符号分隔的键/值对列表。在返回资源之前，Web 服务器可以使用这些参数来执行额外的操作。每个 Web 服务器都有自己关于参数的规则，唯一可靠的方式来知道特定 Web 服务器是否处理参数是通过询问 Web 服务器所有者。

#have-fun 是一个片段标识符，它指示浏览器在渲染页面时滚动到具有相应 ID 的元素。片段标识符不会发送到服务器，因此服务器不会看到它。它只在客户端上处理。可以用作页面内导航

相对url和绝对url

绝对URL包括协议和主机名，而相对URL不包括协议和主机名。相对URL相对于基本URL进行解析。基本URL是包含相对URL的文档的URL。相对URL可以是相对于基本URL的路径，或者是相对于基本URL的路径和查询字符串的组合。具体请查看mdn文档

什么是静态服务器，什么叫动态服务器

静态服务器是指仅提供静态资源的服务器，如 HTML、CSS、JavaScript、图片、视频等。动态服务器是指能够根据请求的不同返回不同内容的服务器，我们称它为“动态”是因为这个应用服务器会在通过 HTTP 服务器把托管文件传送到你的浏览器之前会对这些托管文件如html进行更新。
